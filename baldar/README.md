## 📌 О проекте

**Baldar** («балдар» — «дети» на кырг.) — учётная система для сети детских садов:
управление организациями и филиалами, группами и детьми, посещаемостью,
платёжными кодами и финансами, а также личный кабинет родителя.

Приложение построено как монорепозиторий из двух частей:

| Часть | Назначение |
|---|---|
| `backend` | REST API на Express + Prisma ORM, авторизация по JWT |
| `frontend` | SPA на React (Vite), интерфейс адаптирован под каждую роль |

---

## 👥 Роли и права доступа

| Роль | Возможности |
|---|---|
| `SUPERADMIN` | Полный доступ ко всей системе: организации, платёжные коды, финансы, посещаемость, архив |
| `DIRECTOR` | Управление своей организацией *(раздел в разработке)* |
| `METHODIST` | Методическая работа *(раздел в разработке)* |
| `TEACHER` | Отметка посещаемости своей группы |
| `MEDICAL` | Отметка посещаемости/болезней *(раздел в разработке)* |
| `PARENT` | Просмотр посещаемости и платежей своего ребёнка |

> Проверка прав выполняется как на фронтенде (скрытие разделов), так и на
> бэкенде (`requireRole` в middleware) — фронтенд-проверка служит только для UX.

## 🛠 Технологический стек

**Backend**
- Node.js + Express
- PostgreSQL + Prisma ORM
- JWT (`jsonwebtoken`) — аутентификация
- `bcryptjs` — хеширование паролей
- `zod` — валидация данных

**Frontend**
- React 18 + React Router
- Vite — сборка и dev-сервер
- Tailwind CSS — стилизация
- Axios — HTTP-клиент
- Recharts — графики на дашбордах

## 🚀 Быстрый старт

### 1. Предварительные требования

Убедитесь, что установлено:

- [Node.js](https://nodejs.org/) 18 или новее
- [npm](https://www.npmjs.com/) (идёт вместе с Node.js)
- База данных **PostgreSQL** (локально или в облаке, например [Neon](https://neon.tech), Supabase, Railway)

### 2. Клонирование репозитория

```bash
git clone <URL-вашего-репозитория>
cd baldar
```

### 3. Настройка бэкенда

```bash
cd backend
npm install
cp .env.example .env
```

Откройте `.env` и укажите свои значения:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/kindergarten?schema=public"
JWT_SECRET="сгенерируйте_длинную_случайную_строку"
PORT=4000
```

> 💡 Сгенерировать надёжный `JWT_SECRET` можно командой:
> `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`

### 4. Настройка базы данных

Применить миграции и сгенерировать Prisma Client:

```bash
npm run prisma:migrate
npm run prisma:generate
```

Наполнить базу тестовыми данными (организация, пользователи всех ролей, тестовый ребёнок, платежи, посещаемость):

```bash
npm run seed
```

### 5. Настройка фронтенда

Откройте новый терминал:

```bash
cd frontend
npm install
```

Фронтенд обращается к API по относительному пути `/api` (`src/lib/api.js`), а
dev-сервер Vite сам проксирует эти запросы на `http://localhost:4000`
(см. `server.proxy` в `vite.config.js`). Если бэкенд запущен на другом порту —
поправьте адрес именно там.

### 6. Запуск проекта

В **двух отдельных терминалах**:

```bash
# Терминал 1 — backend (http://localhost:4000)
cd backend
npm run dev
```

```bash
# Терминал 2 — frontend (http://localhost:5173)
cd frontend
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173) в браузере.

## 🔌 API — обзор эндпоинтов

Базовый префикс: `/api`

| Метод | Путь | Описание | Доступ |
|---|---|---|---|
| `POST` | `/auth/login` | Авторизация, выдача JWT | Все |
| `GET` | `/organizations` | Список организаций | `SUPERADMIN` |
| `GET` | `/organizations/:id` | Карточка организации | Авторизован |
| `POST` | `/organizations` | Создать организацию | `SUPERADMIN` |
| `PUT` | `/organizations/:id` | Обновить организацию | `SUPERADMIN` |
| `DELETE` | `/organizations/:id` | Удалить (в архив) | `SUPERADMIN` |
| `GET` | `/payment-codes` | Список платёжных кодов | Авторизован |
| `POST` | `/payment-codes` | Создать платёжный код | Авторизован |
| `PUT` | `/payment-codes/:id` | Обновить платёжный код | Авторизован |
| `GET` | `/finance/summary` | Финансовая сводка | Авторизован |
| `GET` | `/finance/payments` | Список платежей | Авторизован |
| `GET` | `/finance-organizations` | Финансы по организациям | Авторизован |
| `POST` | `/attendance` | Отметить посещаемость | `TEACHER`, `MEDICAL`, `DIRECTOR` |
| `GET` | `/attendance/my-groups` | Группы текущего воспитателя | `TEACHER` |
| `GET` | `/attendance/report` | Отчёт по посещаемости | Авторизован |
| `GET` | `/overview` | Сводные метрики системы | Авторизован |
| `GET` | `/archive` | Список архивных записей | Авторизован |
| `POST` | `/archive/:id/restore` | Восстановить запись из архива | Авторизован |
| `GET` | `/parent/children` | Дети текущего родителя | `PARENT` |
| `GET` | `/parent/children/:childId/attendance` | Посещаемость ребёнка | `PARENT` |
| `GET` | `/parent/children/:childId/payments` | Платежи ребёнка | `PARENT` |
| `GET` | `/health` | Проверка работоспособности API | Все |

Все защищённые маршруты требуют заголовок:
```
Authorization: Bearer <токен>
```

## ⚙️ Переменные окружения

Файл `backend/.env` (создаётся из `.env.example`, **в репозиторий не коммитится**):

| Переменная | Описание | Пример |
|---|---|---|
| `DATABASE_URL` | Строка подключения к PostgreSQL | `postgresql://user:pass@host:5432/db` |

| `PORT` | Порт, на котором поднимается API | `4000` |

## 🔒 Безопасность

- Файл `.env` содержит реальные учётные данные базы данных и **никогда не должен
  попадать в git** — он уже добавлен в `.gitignore`. Коммитьте только `.env.example`.
- Если строка подключения к базе данных когда-либо была закоммичена или передана
  третьим лицам — обязательно смените пароль от БД и `JWT_SECRET`.
- Пароли пользователей хранятся в виде bcrypt-хешей, токены авторизации — JWT
  со сроком жизни 7 дней.
